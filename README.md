# mixture_properties
Вычисление термодинамических и теплофизических свойств смесей газов, жидкостей и твердых веществ на основании данных TERRA и CHEMKIN для отдельных компонент

Логика работы программы:
МАТЕРИАЛ (material.py) состоит из смеси КОМПОНЕНТ (component_mixture.py).
Каждая компонента в общем случае может являться смесью индивидуальных веществ, которые берутся базы данных TERRA (component_terra.py) либо chemkin (component_chemkin.py).
Для жидкости и твердого тела - только из базы TERRA.
Для одного индивидуального вещества возможно использовать только один источник данных, даже если оно представлено в нескольких компонентах (при необходимости это ограничение можно будет снять) - проводятся проверки корректности ввода исходных данных.
Реализовано вычисление транспортных свойств по кинетической теории, для этого данные либо инициализируются из базы данных, либо компонентам присваиваются значения по умолчанию как для воздуха.


# transfer_thermo_C2Q
Конвертор термо файла из CHEMKIN формата в yaml.

Процесс работы примерно следующий - вначале на основании CHEMKIN THERMO файла создается база данных термодинамики. 
В общем случае это может быть очень большая база, в которой далее будут искаться химические компоненты, нужные для реакций.
Далее из подключенного файла реакций выцепляются только имена компонент, которые в наличии.
Далее из базы данных ищутся нужные компоненты и для них вычисляется нужная информация и экспортируется в ямл.
Получаем поля ямл для Ср, газовой постоянной (можно будет поправить на моляруню массу), и для энтальпии образования при 298К.

# transfer_transport_C2Q
Реализовано в составе mixture_properties, но как отдельный скрипт пока в планах. По факту удобно пользоваться через mixture_properties, так как там всё - и теплоёмкость и транспортные свойства - складывается в один файл материала

# chemkin_equilibrium_read
Разбирает excel файл равновесных свойств и состава прореагировавшего газа и создаёт yaml файлы материала. Также вычисляет транспортные свойства по законам смешения с массовыми долями компонент, для этого должны быть заданы транспортные свойства отдельных элементов в базе данных.

# material_verification
Предназначена для визуализации свойств сгенерированных материалов и сравнения их между собой
